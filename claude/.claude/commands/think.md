## 분석 절차

1. **문제 분해**: 핵심 문제를 하위 문제로 나눈다
2. **접근법 비교**: 최소 3가지 다른 접근법을 제시한다
3. **트레이드오프 평가**: 각 접근법의 복잡도, 성능, 유지보수성 비교
4. **리스크 분석**: 엣지 케이스, 잠재적 실패 지점, 하위 호환성
5. **추천**: 최선의 접근법과 그 이유를 명확히 제시

## 출력 형식

각 접근법에 대해:
- 핵심 아이디어 (1-2줄)
- 장점 / 단점
- 구현 복잡도 (낮음/중간/높음)

최종 추천과 구현 순서를 제시한다.
